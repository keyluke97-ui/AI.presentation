비개발자 입장에서 얻은 인사이트 & 꿀팁

1
초기 아키텍처(뼈대)를 그리자

Gemini/GPT 등에 상세히 설명하고 뼈대를 탄탄히.
'만들려는 서비스는 왜 필요할까?', '확장성 고려', '앱/웹 기반' 등

2
무조건 Accept 말고 플래닝 보고 Accept

플래닝 단계를 보고 수락하자!
플래닝 단계를 보지 않고, 무조건적으로 수락하면, 다 만들어지고 또 다시 수정해야해서 시간이 더 걸린다.

3
Following Agent 적극 활용하기!

"해줘"만 하면 나중에 결국에는 복잡한 업무에서 막힌다.
에이전트가 하는 일을 눈으로 보고, 코드가 어떻게 바뀌는지 또는 마우스가 이동하는 방향을 보면 조금씩 업무 속도 향상 및 이해에 도움이 된다.

4
낯선 플랫폼/사이트 – 에이전트와 함께라면 생각보다 괜찮다

비개발자에게 낯선 사이트 플랫폼(github,vercel)도 에이전트와 같이 쓰면 부담이 덜하다.
사이트를 돌아다니는 모습을 구경하면, 구성이 눈에 익기 시작합니다.

5
에이전트와 함께라면 타 서비스와 연동도 어렵지 않다!

API는 생각보다 많이 열려있다!
에어테이블, 비즈뿌리오, 샵바이, 제미나이 등.

6
모르는 용어는 비개발자를 위한 개발자 용어 설명해주는 GEMs·GPTs를 만들어서, 그때 그때 배워놓자!

비개발자 기준으로 그때그때 쉽게 설명해달라고 요청해서 만들어 두자.
나오는 단어는 결국에는 또 나오고, 추후에는 간단한 수정은 comment 기능으로 간단하게 수정할 수 있다!

7
비교적 명확한 반복 업무 자동화는 쉽다!

DB만 정교하게 쌓여있다면, 명확한 정보를 프레임워크 위에 띄워주는 건 바이브 코딩으로 큰 오류 없이 할 수 있다!
e.g) 크리에이터 프리미엄 협찬 신청 페이지

8
제미나이를 붙여보자! (feat. gemini-3-flash)

특히 gemini-3-flash 모델은 일일 1000~1500회 무료 API 호출 가능!
* 특정 웹페이지 긁어서 Gemini로 분석 후 Slack/문자/이메일 전송
* DB 업데이트 및 특이사항 분석 자동화 등 활용도 무궁무진

9
안 되는 게 없다고 생각하면 좋다

서버를 관리하기는 어렵겠지..? → 서버리스 배포 → 이미지 저장 허들 →
알고보니, 쉬운 문제. 안 되는 게 없다고 생각하자!

Tip 1 예시

초기 아키텍처(뼈대) 그리기 – 실제 프롬프트 예시

AI에게 물어보기 전, 무엇을 만들지 상세히 설명하고 뼈대를 탄탄히 세우는 것이 중요합니다.

예시 AI 사진 분석 서비스 – 초기 프롬프트
🏕️ 캠핏(Camfit) AI 그로쓰 에디터

서비스 개요

  • 목적: 캠핑장 사진 업로드 → AI(Gemini)가 '캠핏 A급 표준'에 따라 분석
  • 타겟: 캠핑장 운영자 및 관리자

핵심 UI 구성 요소

  • Hero Section: 서비스 슬로건, 가치 제안
  • Data Input: 캠핑장 정보 입력, 사진 업로드 UI
  • AI Analysis Report: 종합 점수, 4대 지표 카드

기술적 요구사항

  • Framework: React / Next.js
  • Responsiveness: PC·모바일 반응형
⚠️ 주의 모바일 고려 안 하면?
📱 모바일 미고려 시 발생한 문제

💡 핵심 포인트: 만들려는 서비스가 왜 필요한지, 확장성은 어떻게 고려할지, 앱/웹 기반 중 무엇으로 할지 등을 먼저 정리하자!
